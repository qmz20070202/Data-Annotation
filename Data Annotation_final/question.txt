 一、UI/UX 视觉与交互设计优化
自定义交互弹窗设计

摒弃浏览器默认弹窗：所有交互弹窗（如确认、提示、警告）都应替换为应用内自定义设计的模态框（Modal），确保视觉风格统一。弹窗应在屏幕中央显示。

文件上传确认：当用户选择文件或文件夹准备上传时，弹出一个自定义弹窗，询问“您确定要上传 [文件夹名称] 吗？”，包含“确认上传”和“取消”按钮。

重复操作提示：当用户对一张已经完成识别的图片再次点击“识别”按钮时，弹窗提示：“该图片已处理完毕，无需重复操作。”

整体美学风格提升

按钮与色彩：按钮设计应更加醒目、鲜艳，具有清晰的悬停（Hover）和点击（Click）效果。整体配色方案需遵循现代美学原理，确保色彩和谐、视觉舒适。

构图与布局：界面元素（如卡片、按钮、输入框）的边角建议采用更圆润的设计（增加 border-radius），营造柔和、亲切的视觉感受。

🗂️ 二、核心功能：文件夹持久化存储
实现浏览器端持久化：用户上传的文件夹数据必须被保存在浏览器中。即使用户关闭或刷新网页后重新打开，之前上传的所有文件夹及其处理状态都应完整保留，无需重新上传。

技术建议：可使用 IndexedDB 或 File System Access API 等现代浏览器技术来实现稳定、高效的本地数据持久化。

📑 三、页面结构与核心工作流
应用需包含两个主要页面：主控台（Dashboard） 和 校准页（Calibration View）。

1. 页面一：主控台 (Dashboard)
这是应用的入口和管理中心。

布局设计：主控台页面采用三栏式布局，清晰地展示不同状态的文件夹：

第一栏：未识别 (Unprocessed)：所有新上传的文件夹默认进入此栏。

第二栏：已识别 (Processed)：已完成批量识别，但尚未校准的文件夹存放于此。

第三栏：已校准 (Calibrated)：所有图片均已完成人工校准并保存的文件夹存放于此。

核心工作流：

上传：用户在主控台点击“上传文件夹”按钮，上传的文件夹作为一个卡片出现在“未识别”栏中。

批量识别：用户可以从“未识别”栏中选择一个或多个文件夹，点击“开始批量识别”按钮。

状态自动流转 (识别 -> 校准)：当一个文件夹内的所有图片被后台识别完毕后，该文件夹卡片将自动从“未识别”栏移动到“已识别”栏，页面状态实时更新。

进入校准：用户点击“已识别”栏中的任意一个文件夹卡片，页面将跳转至校准页，并加载该文件夹内的所有图片以供校准。

2. 页面二：校准页 (Calibration View)
此页面用于对单个文件夹内的图片进行逐一的人工校准。

页面功能：此页面与现有工具的功能相似，用户可以查看图片、修改识别结果并保存为 JSON 数据。

核心工作流：

校准操作：用户在此页面对文件夹内的图片进行检查和校准。

完成校准：当用户确认该文件夹内所有图片都已校准无误后，点击右下角的 “全部完成” 按钮。

状态自动流转 (校准 -> 完成)：点击“全部完成”后，执行两个操作：

页面自动跳转回主控台。

刚刚完成校准的文件夹卡片，将自动从“已识别”栏移动到“已校准”栏。
